# PRD: Matrix 客户端产品形态（用户无感 OpenClaw）

## 1. 目标
构建“用户仅感知数字员工协作”的客户端体验：
- 用户通过 Matrix 客户端（Web/桌面/移动）进行沟通与操作。
- OpenClaw 作为租户实例运行时，仅在平台侧承载执行，不在用户端暴露。
- 管理后台对实例、资产、日志、权限进行治理与审计。

## 2. 客户端形态

### 2.1 用户端（C 端）
- Web: Element Web（已部署）
- Desktop: Element Desktop（企业分发）
- Mobile: Element iOS / Android（标准 Matrix 登录）

### 2.2 管理端（B 端）
- Web 管理后台（已上线）：
  - 总览
  - 员工管理（实例视角）
  - 技能管理（共享资产）
  - 工具管理（MCP 服务）
  - 行为日志
  - 账号权限

### 2.3 平台隐形层
- Matrix 为主渠道。
- 控制面负责：实例编排、共享资产流转、审计落库。
- OpenClaw 为执行引擎（每租户独立实例）。

## 3. 核心用户旅程

### 3.1 创建数字员工
1. 用户在 Matrix 房间发送 `!create_agent <name>`。
2. 系统创建租户实例并回传数字员工卡片。
3. 用户点击卡片进入会话窗口继续任务。

### 3.2 日常协作
1. 用户在房间 `@数字员工` 下发任务。
2. Agent 执行并在会话反馈结果。
3. 管理端可看到实例状态、事件日志与资产沉淀。

### 3.3 治理与追溯
1. 管理员在后台查看异常实例和待审批资产。
2. 对共享资产执行审批/发布/绑定。
3. 通过日志与权限追溯具体操作责任。

## 4. 能力边界

### 4.1 当前版本（必须）
- Matrix 渠道创建实例闭环。
- 管理后台六大模块可用。
- 共享资产审批与绑定可用。
- 审计日志可检索。
- 角色权限可配置。

### 4.2 后续版本（可选增强）
- 桌面端企业安装包自动配置 homeserver。
- 会话内执行状态条与失败重试入口。
- 通知中心聚合失败/审批待办。

## 5. 非功能要求
- 租户隔离：实例、存储、配置独立。
- 可观测性：实例状态 + 审计事件 + 资产流转可追踪。
- 一致性：Web/桌面/移动端交互语义一致。
- 安全性：RBAC、审计留痕、最小权限。

## 6. 成功指标
- 用户成功创建数字员工的完成率。
- 创建后 5 分钟内首次有效对话率。
- 后台待审资产清理时延。
- 实例异常恢复时长（重建/重启）。
