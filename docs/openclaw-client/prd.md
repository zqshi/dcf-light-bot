# OpenClaw Client Suite — 产品需求文档 (PRD)

> 版本 1.0 | 2026-04-14 | MVP 阶段

## 1. 产品定位

OpenClaw Client Suite 将 AI 协作从"工具模式"升级为"管理者模式"——用户做决策，AI 自主执行。

| 维度 | 工具模式（旧） | 管理者模式（新） |
|------|-------------|---------------|
| 用户角色 | 操作者，逐条指令 | 决策者，定义目标和边界 |
| AI 角色 | 被动执行器 | 自主执行者，主动汇报 |
| 信息流 | 指令→结果（线性） | 目标→进展→决策点→结果（异步闭环） |
| 效率瓶颈 | 人的操作速度 | 人的决策质量 |

## 2. 用户角色

| 角色 | 职责 |
|------|------|
| **管理者** | 设定目标、审批决策、确认方向 |
| **数字员工（AI Agent）** | 自主执行、主动发现、生成推荐方案 |

## 3. 三层授权模型

```
Layer 3: 自主执行（人不介入）— 低风险、可逆、有模板
Layer 2: 执行 + 汇报（异步知会）— 中风险、需人知晓
Layer 1: 决策请求（同步等待）— 高风险、不可逆、涉外部承诺
```

## 4. 核心能力（MVP）

### 4.1 AI 对话

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 对话消息流 | 富文本对话，支持 Markdown 渲染 | 消息实时显示，支持代码块/链接/图片 |
| CoT 推理链 | AI 思考过程可视化（工具调用/知识引用） | 展开可见推理步骤 |
| 多 Agent 路由 | 按能力自动路由到合适的 Agent | 路由透明，可切换 |
| Mock 演示模式 | 完整 mock 数据支撑离线演示 | 全功能可用，无后端依赖 |

### 4.2 异步决策流 (P0)

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 决策请求卡片 | AI 推荐方案 + 理由 + 操作按钮 | 嵌入对话流，一键操作 |
| 决策详情面板 | Drawer 内展示完整决策上下文 | 推荐/备选方案对比 |
| 紧急度标识 | critical/high/normal/low 四级 | 视觉区分 + 时效倒计时 |
| 决策树 | 复杂决策分支可视化 | 节点展开/折叠 |

### 4.3 目标驱动 (P1)

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 目标设定 | Composer 内自然语言设定目标 | AI 自动拆解为里程碑 |
| 里程碑追踪 | 进度百分比 + 状态标记 | Drawer 内目标追踪面板 |
| 进展汇报 | AI 主动汇报里程碑完成 | 嵌入对话流 |
| 决策触发 | 里程碑完成时触发确认决策 | DecisionHub 统一入口 |

### 4.4 执行看板 (P2)

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| Agent 工作台 | 所有数字员工实时状态 | 运行/等待决策/空闲/异常 |
| 协作链路 | 跨 Agent 任务委托可视化 | 节点状态 + 进度百分比 |
| 项目看板 | 四栏面板（任务/目标/决策/协作） | 全景视图 |

### 4.5 知识库

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 文档管理 | 文件夹/文档 CRUD | 树形导航 |
| 富文本编辑 | TipTap 编辑器 | 支持图片/链接/格式 |
| 版本管理 | 版本历史 + 回滚 | 版本对比 |
| 权限控制 | 文档级权限 + 审批 | 权限审批流 |
| 代码编辑 | CodeMirror 多语言 | JS/TS/Python/JSON/CSS/HTML/Markdown |

### 4.6 应用中心

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 应用模板 | 预置应用模板 | 一键创建 |
| 应用构建器 | 对话式应用构建 | AI 辅助生成 |
| 文档编辑器 | 应用内嵌文档 | TipTap |

### 4.7 通知与待办

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 跨渠道通知 | 聚合 Matrix/Lark/Slack 等通知 | 统一列表 |
| 通知详情 | 完整上下文 + 快捷回复 | Drawer 面板 |
| 待办管理 | Todo 列表 + 优先级 | 拖拽排序 |

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏 <3s，对话消息 <500ms 渲染 |
| 离线 | Mock 模式可完整演示 |
| 无障碍 | 键盘导航、语义化标签 |
| 响应式 | 桌面优先，平板适配 |
| 类型安全 | TypeScript strict 模式，no any |
| 测试 | domain 层 100% 覆盖，15 个测试文件 |

## 6. 设计规范

- Apple HIG glass morphism 设计语言
- 主色 `#007AFF`，圆角 `14px`，毛玻璃 `backdrop-filter: blur`
- 暗色模式通过 CSS 变量切换
- Tailwind CSS 原子类，`@dcf/ui-tokens` 统一 token
